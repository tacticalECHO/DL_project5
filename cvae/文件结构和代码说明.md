## 二、代码文件说明

### 2.1 `model.py` — 模型定义

包含 4 个 `nn.Module` 类：

| 类 | 作用 | 参数量 | 输入 | 输出 |
|---|---|---|---|---|
| `Encoder` | 后验网络 $q_\phi(z \mid x, y, M)$ | 23.74M | `concat(gt, masked, mask)` 7 通道 | $\mu_q, \log\sigma_q^2 \in \mathbb{R}^{256}$ |
| `PriorNet` | 条件先验 $p_\psi(z \mid y, M)$ | 9.95M | `concat(masked, mask)` 4 通道 | $\mu_p, \log\sigma_p^2 \in \mathbb{R}^{256}$ |
| `Decoder` | 生成网络 $p_\theta(\hat{x} \mid z, y, M)$ | 15.71M | $z$ + `masked` + `mask` | $\hat{x} \in [0,1]^{3 \times 256 \times 256}$ |
| `CVAE` | 顶层封装，把三者串起来 | 49.40M | `(gt, masked, mask)` | $(\hat{x}, (\mu_q, \log\sigma_q^2, \mu_p, \log\sigma_p^2))$ |

**关键设计**：
- Decoder 采用 **Upsample + Conv**（避免 ConvTranspose 的棋盘伪影）
- Decoder 每个上采样层级都 concat 下采样后的 `(masked, mask)` 作为空间条件
- **没有 U-Net skip connection**，保证 $z$ 真正承载信息
- **不做 compose**（不做 `masked + x_raw * mask`），decoder 直接输出整张图。这是 v1.3 相对 v1.2 的关键修改。

**独立运行**：`python cvae\model.py` 会跑完整 shape 和梯度流自测。

### 2.2 `loss.py` — 损失函数

核心函数 `cvae_loss`：

$$
\mathcal{L}(x, c) = \|\hat{x} - x\|_1 + \beta(t) \cdot D_{KL}\big(q_\phi(z \mid x, c) \,\|\, p_\psi(z \mid c)\big)
$$

其中 KL 项是两个对角高斯的闭式解，用 `kl_gauss_gauss` 函数计算。

**辅助函数 `beta_schedule`**：$\beta$ 在前 10 个 epoch 从 0 线性升到 1.0，之后保持。

**保留的历史函数**（未使用但留作 ablation 参考）：
- `kl_standard_normal`：v1.2 用的 $D_{KL}(q \| \mathcal{N}(0, I))$
- `kl_standard_normal_free_bits`：带 free bits 下限的版本

独立运行：`python cvae\loss.py` 跑 8 个单元测试（KL 正确性、不对称性、梯度流等）。

### 2.3 `train.py` — 训练循环

主流程：加载数据 → 实例化 CVAE → Adam + AMP 混合精度训练 → 每 epoch 存 checkpoint 和可视化。

**关键配置**（默认值）：
- batch size = 12
- lr = 1e-4
- epochs = 50
- warmup_epochs = 10，max_beta = 1.0
- 训练 mask 分布：三档 union（`regime="mixed"`）
- 数据增强：水平翻转 p=0.5

**输出**：
- `checkpoints/{run_name}/last.pt` 和 `best.pt`（按 val L1 最低选 best）
- `checkpoints/{run_name}/train.log`（CSV 格式训练日志）
- `checkpoints/{run_name}/vis/epoch_*.png`（每 epoch 的 4 样本可视化）

### 2.4 `eval.py` — 测试集评估

功能：加载训练好的 `best.pt`，在三档 test fixed-mask 数据集上计算 5 个指标：MSE / PSNR / SSIM / LPIPS / FID。

**输出两个文件**（默认写到项目根的 `reports/` 目录，已手动移到 `cvae/reports/`）：

1. **Per-image CSV**：每张 test 图一行
   ```
   path, regime, mask_ratio, mse, ssim, psnr, lpips
   ```
   共 9000 行（3000 test 图 × 3 regime）。用于 per-image 级分析（如异常值定位、方差计算）。

2. **Summary JSON**：每档 mask 一条
   ```json
   {"method": "cvae", "regime": "small", "mse": ..., "ssim": ..., "psnr": ..., "lpips": ..., "fid": ...}
   ```
   用于 P-D 平面和 report 总结表。

**注**：FID 必须整批累积后一次性计算（分布距离无法 per-image），故只在 summary 中出现。

### 2.5 `visualize.py` — 可视化

两个功能：

**(1) Pluralistic grid**：对同一张 masked 图用不同 $z$ 采样多次，展示 CVAE 的随机性。每行：`gt | masked | s1 | s2 | ... | sN`。

**(2) P-D 平面图**（CVAE 单独版）：两个子图
- 左：MSE vs FID
- 右：(1-SSIM) vs LPIPS
- 三档 mask 为三个点，连线观察趋势

后续组员汇合时需要的"三方法联合 P-D 平面图"尚未生成，需等 Recon 和 GAN 组员的 `summary.json` 合并后单独做。

## 四、如何快速复现

### 前置条件
- 已经跑过根目录的 `split_paths.py` 和 `make_fixed_masks.py`，生成 `data/{train,val,test}_paths.txt` 和 `data/fixed_masks_*.pt`
- 已经安装 `requirements.txt`（见项目根）
- GPU 建议显存 ≥ 10GB（batch 12 下约占 8GB）

### 完整复现三条命令

```bash
# 1. 训练（约 4-5 小时 on RTX 2080 Ti）
python cvae/train.py --epochs 50 --run-name run_01 \
    --batch-size 12 --num-workers 2

# 2. 评估（约 10 分钟）
python cvae/eval.py --ckpt checkpoints/run_01/best.pt \
    --run-name run_01 --batch-size 16

# 3. 可视化（约 1 分钟）
python cvae/visualize.py --ckpt checkpoints/run_01/best.pt \
    --regime large --n-images 4 --n-samples 6
```

### 只想看模型结构（不跑训练）

```bash
python cvae/model.py    # 走一遍 shape 和梯度测试
python cvae/loss.py     # 跑 loss 函数单元测试
```

## 六、方案演化简史（可写进 report Discussion）

| 版本 | 设计 | 结果 |
|---|---|---|
| v1.2 | 标准先验 $\mathcal{N}(0, I)$ + compose | **失败**：posterior collapse，KL 断崖到 0，val loss 不降 |
| v1.2 + free bits | 加 KL 下限 | **仍失败**：保住了 posterior 容量，但 decoder 学会无视 $z$ |
| **v1.3** | **条件先验 $p_\psi$ + 无 compose** | **成功**：val L1 稳定下降，medium FID = 7.93 |

关键教训：compose 操作（只重建 hole）让 decoder 可以忽略 $z$，必须去除；固定先验下 KL 优化会硬拉 posterior，导致 collapse，必须用条件先验。这实际上重现了 Pluralistic Image Completion (Zheng et al. CVPR 2019) 的核心动机——**conditional prior 不是可有可无的装饰，而是防止 collapse 的必要设计**。

完整迭代记录见 `reports/cvae_method.md` §3。

## 七、常见问题

**Q: 训练中看到 `torch.load weights_only` 的 warning？**
A: 来自 `utils/dataset.py`（组内共享代码），加载 fixed_masks 时触发。不影响运行。后续统一清理。

**Q: 生成的人脸在眼睛、嘴附近看起来模糊？**
A: 这是 L1 重建损失的固有特性——L1 鼓励所有合理补全的像素均值，对高频细节天然偏模糊。不是 bug。FID 和 LPIPS 指标不受此影响（仍然不错）。如需消除模糊，需要加对抗损失或 perceptual loss，但会违反"non-GAN generative"的方法定位。

**Q: `check_pluralistic.py` 在哪？**
A: 在项目根的 `scripts/` 目录下（不在 `cvae/` 里）。这是一个诊断小脚本，不是正式产物。

**Q: 如果要重新生成 fixed_masks，会影响我的评估吗？**
A: 会！fixed_masks 是三人共享的评估 mask。如果重新生成（不同 seed），三方法结果就不可比。**不要动 `data/fixed_masks_*.pt`**。如果确需重生成，务必通知组员同步重跑 eval。
